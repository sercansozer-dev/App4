Overview

NOTE: For an optimal experience, please view this documentation in Mozilla Firefox.

The GoPxL SDK is a C++ API library that supports communication with GoPxL sensors and systems.

The GoPxL SDK is an open-source library provided under the MIT license.

Sample Applications

The SDK includes 18 sample applications demonstrating various SDK features:

  • Sensor configuration and control
  • Data reception (profiles, surfaces, measurements, images)
  • Multi-sensor layouts and alignment
  • Job management and replay functionality
  • Network discovery

Sample source code is located in the samples/ directory.

Building the SDK

A generate.py script is included in the SDK that generates platform-specific build files for the SDK library and all sample applications:

  • Visual Studio solution/project files for Windows (VS2017, VS2019, VS2022)
  • GNU Makefiles for Linux x64

Generating Build Files

Run the generate script from the GoPxLSdk directory:

> cd *GoPxL_SDK/Gocator/GoPxLSdk
> python -m generate

Windows Build

After running generate.py, open GoPxLSdk.sln in Visual Studio 2017 or later and build the solution. This will build the SDK library and all sample applications.

Linux Build

After running generate.py to create the Makefiles, build the SDK and samples:

> cd *GoPxL_SDK/Gocator
> make -f GoPxLSdk-Linux_X64.mk

Compiled output locations:

  • SDK library: *GoPxL_SDK/lib/linux_x64/libGoPxLSdk.so
  • Sample executables: *GoPxL_SDK/bin/linux_x64/

System Requirements

Tested Operating Systems:

  • Windows 10 (64-bit)
  • Ubuntu 18.04, 20.04, 22.04 (64-bit)

Compiler Requirements:

  • Visual Studio 2017 or later (Windows)
  • GCC compiler (Linux)

Python Requirements:

  • Python 3.7+ (for running generate.py)

Units

The GoPxL SDK uses mm, mm2, mm3, hertz, microseconds, and degrees as standard units unless otherwise mentioned.

Important Usage Notes

Before creating any SDK objects, you must first construct the GoPxL API core framework by calling GoApiLib_Construct().

For detailed usage examples and API demonstrations, refer to the 18 sample applications included with the SDK.
